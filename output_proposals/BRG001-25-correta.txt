CLIENTE: Bridgestone  
PROJETO: Sistema de Coleta e Relatórios de Dados de Produção  
DATA: [Data Atual]

----------------------------------------------------------------------
1. OBJETIVO DO PROJETO
----------------------------------------------------------------------
Desenvolver um sistema que coleta dados de cinco máquinas de produção de pneus a cada cinco segundos, armazena essas informações em um banco de dados SQL Server e permite a visualização dos resultados em relatórios tabulares.

----------------------------------------------------------------------
2. FUNCIONALIDADES DA SOLUÇÃO COMPLETA
----------------------------------------------------------------------
- **Módulo de Coleta de Dados:**
  - Comunicação automática com máquinas de produção via protocolos industriais.
  - Suporte para até 300 tags de dados.
  - Tratamento de falhas e logs de monitoramento.

- **Módulo de Armazenamento de Dados:**
  - Banco de dados SQL Server 2019 otimizado com índices e particionamento.
  - Rotinas de limpeza periódica para gerenciamento de volume de dados.

- **Módulo de Relatórios:**
  - Desenvolvimento de até 12 relatórios em SSRS para consumo dos usuários finais.

- **Módulo de Testes Integrados:**
  - Testes de coleta diretamente nas máquinas (in loco).

- **Módulo de Apoio ao Consumo e Entendimento dos Dados:**
  - Treinamento de usuários para uso da interface e interpretação dos dados.
  - Disponibilização de material de apoio.

- **Módulo de Go Live e Suporte Inicial:**
  - Participação no Go Live e suporte inicial aos usuários.

5. PREMISSAS E RESTRIÇÕES
----------------------------------------------------------------------
**Premissas:**
- Acesso contínuo às máquinas de produção para integração.
- Disponibilidade de infraestrutura de TI para instalação do SQL Server e SSRS.
- Participação ativa dos usuários finais para definição de requisitos e testes.

**Restrições (Fora do Escopo):**
- Integração com sistemas externos não especificados no briefing.
- Desenvolvimento de funcionalidades além dos 12 relatórios SSRS.

----------------------------------------------------------------------
6. SUGESTÃO DE MVP (MÍNIMO PRODUTO VIÁVEL)
----------------------------------------------------------------------

Para gerar valor rapidamente e mitigar riscos, propomos uma entrega inicial focada no maior impacto de negócio.

**Módulo(s) / Funcionalidade(s) do MVP:**
- Coleta de Dados de duas máquinas críticas (Triplex A e Calandra 1).
- Armazenamento básico no SQL Server.
- Desenvolvimento de 3 relatórios SSRS essenciais para monitoramento inicial.

**Justificativa de Valor (Efeito UAU):**
- Este MVP é usável pois resolve o problema imediato de coleta e visualização de dados críticos, viável por ser implementado rapidamente, e gera alto valor percebido ao permitir que a Bridgestone comece a monitorar e otimizar a produção com dados reais.

**Estimativa de Esforço e Tempo para o MVP:**
- **Horas:** 240
- **Cronograma:** 3 semanas

**Evolução Pós-MVP:**
- Após o sucesso do MVP, os demais módulos e funcionalidades serão integrados em fases subsequentes, expandindo a coleta para todas as máquinas e completando o conjunto de relatórios planejados.
